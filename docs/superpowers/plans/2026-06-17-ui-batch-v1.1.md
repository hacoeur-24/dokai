# v1.1.0 UI Batch Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.

**Goal:** A batch of UI/UX features for the dokai-ui app, shipped in the same v1.1.0 as the OpenAPI explorer: a full-width site header (GitHub/App links + search), a richer create-new flow (folder/document + real empty folders), sidebar create + collapse-all controls, a custom Dropdown + InfoTip, search filters, wider content, and i18n fixes.

**Architecture:** Mostly dokai-ui edits + two new shared components (`Dropdown`, `InfoTip`) built on the already-installed `@radix-ui/react-dropdown-menu`, plus a small `dokai-core` settings-schema addition (`githubUrl`/`appUrl`) and one new `/api/folder` middleware route. Layout becomes a vertical stack: a full-width `AppHeader` over the existing sidebar+main row.

**Tech stack:** React 19, react-router, `@radix-ui/react-dropdown-menu@2.1.4`, `lucide-react@0.469` (`Github`, `SquareMousePointer`, `Info`, `Plus`, `ChevronDown`, `ChevronsDownUp`/`ChevronsUpDown`, `SlidersHorizontal`), `cmdk`, Zod, Vitest (node env).

## Global Constraints

- Node ≥ 22; pnpm 10.x. **No new dependencies** — build the Dropdown/InfoTip on the existing `@radix-ui/react-dropdown-menu` and the existing `.dokai-dropdown-menu`/`.dokai-dropdown-item` CSS.
- ESM only; `.js` extensions on relative imports. TS strict (`noUncheckedIndexedAccess`, `verbatimModuleSyntax` → `import type`, `useUnknownInCatchVariables`). No emojis in source.
- **i18n is mandatory for all new user-facing strings.** Keys are flat dot-strings in `packages/ui/src/i18n/locales/en.json` and `fr.json`; access via `useT()` → `t(key, vars)`. Every new EN key gets a FR counterpart. Never hardcode a visible string.
- ui Vitest is pure-Node, `*.test.ts` only (no jsdom). Unit-test pure logic (schema, tree-flatteners, pure helpers); verify React components via `pnpm --filter dokai-ui typecheck` + the final e2e smoke. Never add a `.test.tsx`.
- Theme: all colors are `var(--color-*)`; `[data-theme='dark']` overrides on `<html>`. New CSS must use tokens so it themes automatically. The InfoTip tooltip is the one INVERTED case (dark bubble on light theme, light on dark).
- `dokai-core` settings schema is published in lockstep (already at 1.1.0).
- Decisions (confirmed with the user): **full-width header** across the top (restructure Layout to a column); **real empty folders** via `POST /api/folder` writing `_section.json`; header search is a **trigger to the existing SearchPalette** (no second search); content width **76ch**; **item-8 port change is OUT of scope** (it lives in the user's mavira repo, not DOKAI); the diagram buttons to translate are the REAL ones (`Copy source`/`Copied!`/`SVG`/`PNG`/`Close` + `Diagram` eyebrow + zoom titles).
